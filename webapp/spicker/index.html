Welches Werkzeug für welchen Use Case — verifizierte Grenzen, Hinweise & End-User-Lizenz

Use Case Empfohlenes Werkzeug End-User-Lizenz Verifizierte Grenze / Hinweis
Nur Web-Recherche, Drafting – keine Work-/Tenant-Daten M365 Copilot Chat (ohne Lizenz) Keine Zusatzlizenz
Eligible M365-/O365-Abo mit Entra-Anmeldung genügt (web-grounded).		 Geeignet für öffentliche Web-Recherche und Drafting. Ohne Add-on kein vollständiges Work-IQ-/Graph-Grounding; geöffnete oder hochgeladene Dateien und begrenztes Outlook-Grounding sind Ausnahmen. Achtung Laut Message Center MC1253858 ist Copilot Chat seit 15.04.2026 in Tenants >2’000 Seats für unlizenzierte User nicht mehr in Word, Excel, PowerPoint und OneNote verfügbar. Diese tenantbezogene Änderung fehlt weiterhin in der öffentlichen Learn-Dokumentation.
Persönliches Work-Grounding: «Schau in meinen letzten Mails nach» – eigene Mails, Kalender, OneDrive, Meetings M365 Copilot (Work-Tab / Office-Apps) M365 Copilot Lizenz zwingend
Nicht über PAYG oder Credits abbildbar.		 Verifiziert 2026: Vollständiges Work IQ, semantische Suche und breites persönliches Graph-Grounding benötigen die User-Lizenz und lassen sich nicht über PAYG ersetzen. Ohne Add-on bietet Outlook offiziell begrenztes Grounding auf Inbox, Kalender, Meetings und bestimmte Dateien. PAYG kann Agenten auf vordefinierten, geteilten SharePoint- oder Connector-Quellen ermöglichen, aber kein vollständiges persönliches Work Grounding.
Tiefe Web-/Work-Recherche mit Microsoft Researcher Researcher-Agent / M365 Copilot M365 Copilot Lizenz
Nativer Agent (wie Analyst, Facilitator): für Lizenzierte ohne Zusatz-Credits, für Unlizenzierte nicht via Credits freischaltbar.		 Researcher ist eine Capability innerhalb von Microsoft 365 Copilot. Sichtbarkeit hängt von Lizenzierung, Rollout und Admin-Konfiguration ab.
Copilot direkt auf SharePoint-Site / Dokumenten-Q&A SharePoint Agent (ready-made) Lizenz oder Credits
Für M365-Copilot-lizenzierte User ohne zusätzliche Copilot Credits enthalten; sonst PAYG/Capacity Packs des Tenants.		 Jede SharePoint-Site hat einen ready-made Agent auf Basis der Site-Inhalte. Nutzung erfordert M365 Copilot Lizenz oder aktivierten Pay-as-you-go-Service.
SharePoint-Agent mit eigener Quellen-Auswahl SharePoint Agent (custom) Lizenz oder Credits
Wie ready-made: M365 Copilot oder PAYG/Packs.		 Bis zu 20 Source Items (Sites, Dokumentbibliotheken, Ordner oder Dateien). Bei mehr als 20 Quellen höheres Nesting nutzen – ganze Site/Library statt einzelner Dateien.
Mehrere Sites / grössere Wissensbasis / viele Knowledge Sources Copilot Studio Lizenz oder Credits
Bei authentifizierter B2E-Nutzung unter der Identität eines M365-Copilot-lizenzierten Users ohne Zusatz-Credits enthalten; sonst Tenant-Credits (Packs/PAYG).		 Bis zu 500 Knowledge Sources pro Agent über alle Typen hinweg.
Eine einzelne SharePoint-/Microsoft-Liste als Knowledge Source M365 Agent Builder / SharePoint Custom Agent oder Copilot Studio Lizenz oder Credits
M365 Copilot Lizenz oder PAYG.		 Update 2026 / Roadmap 559022: Max. 1 SharePoint-Liste pro Agent mit höchstens 20’000 Elementen oder 50 MB Rohtext. Listenanhänge werden nicht indexiert. Eine Site-URL schliesst deren Listen nicht ein; die konkrete Liste muss als Quelle gewählt werden. Für mehrere Listen → Copilot Studio (bis 15). Persönliche OneDrive-Agents sind ein separates Szenario und unterstützen kein PAYG.
Mehrere SharePoint-Listen Copilot Studio Lizenz oder Credits
Authentifizierte B2E-Nutzung unter der Identität eines M365-Copilot-lizenzierten Users ist ohne Zusatz-Credits enthalten; sonst Credits.		 Bis zu 15 Listen pro Agent (empfohlen max. 10). Max. 35’000 Rows pro Liste und insgesamt max. 120’000 Rows; grössere Listen beeinträchtigen Qualität und Latenz.
Verlässliche Counts, Summen, Durchschnittswerte oder strukturierte Abfragen Copilot Studio + Action / Flow / API Lizenz oder Credits
Abrechnung hängt von Action, Flow-Trigger und Benutzerlizenz ab.		 Für deterministische Aggregationen eine Action, OData-/Dataverse-Abfrage, API oder einen Flow verwenden. SharePoint Lists Knowledge allein ist dafür ungeeignet: Pro Listenabfrage werden nur die ersten 2’048 Zeilen berücksichtigt und es wird kein deterministischer Analysecode ausgeführt.
Filtern, OData-Abfragen, gezielte Queries, Schreiben in Listen/Systeme Copilot Studio + Actions / Agent Flow / Power Automate / Connector Lizenz oder Credits
Regulär 13 Copilot Credits pro 100 Agent-Flow-Actions. Ohne Zusatz-Credits enthalten, wenn When an agent calls the flow durch einen authentifizierten M365-Copilot-lizenzierten User ausgelöst wird; andere Trigger verbrauchen Credits. Power-Automate-Cloud-Flows werden separat lizenziert.		 Für gezielte Abfragen, Business-Logik, Writebacks oder CRUD-Szenarien Actions, Agent Flows, Power Automate oder Connectors nutzen. Knowledge Sources allein sind dafür nicht der richtige Mechanismus.
Persönliche Files/Folders OneDrive Agent M365 Copilot Lizenz zwingend
Erstellung und Nutzung persönlicher OneDrive-Inhalte sind nicht über Consumption Billing/PAYG verfügbar.		 Für persönliche Dateien und Ordner in OneDrive. Der Agent wird als .agent gespeichert. Max. 20 Quellen; ein gewählter Ordner darf höchstens 20 unterstützte Dateien enthalten. Geteilte User benötigen zusätzlich Zugriff auf die enthaltenen Dateien.
Teams / Outlook / Web / externe Daten als Knowledge M365 Copilot Agent Builder / Declarative Agent Je nach Quelle
Instructions und öffentliches Web: ohne Zusatzkosten. SharePoint, Embedded Files, Dataverse oder Copilot Connectors: M365 Copilot oder Metering. E-Mail, People, Teams-Nachrichten und Meetings: M365 Copilot Lizenz, nicht PAYG.		 Für einfache, deklarative Agenten innerhalb von M365 Copilot. Die Knowledge Source bestimmt die Berechtigung: persönliche Outlook-/Teams-Daten benötigen das Add-on, während geteilte Tenant-Daten teilweise über Copilot Credits bereitgestellt werden können.
Cross-System, Dataverse, Power Automate, APIs, komplexe Workflows Copilot Studio (full) Lizenz oder Credits
Maker benötigen eine zugewiesene Copilot Studio User License sowie ein Tenant-Entitlement mit Kapazität/PAYG. Authentifizierte B2E-Nutzung durch M365-Copilot-lizenzierte User ist im dokumentierten Umfang ohne Zusatz-Credits enthalten; sonst Credits.		 Richtige Wahl, wenn der Agent über mehrere Systeme arbeitet, Actions ausführt, Dataverse/Power Automate/API-Integrationen benötigt oder produktiv für grössere Zielgruppen bereitgestellt wird.
Autonome oder event-getriggerte Agents Copilot Studio / Custom Engine Agent Credits immer
Autonome Runs sind nie zero-rated – auch nicht für M365-Copilot-Lizenzierte.		 Für automatische Aktionen, Trigger, Agent Flows und komplexe Automatisierungen. Billing und Credits separat gemäss aktueller Microsoft-Dokumentation prüfen.
Externe Website- / App-Kanäle Copilot Studio Copilot Credits
Anonyme und externe Benutzer sind nicht durch eine M365-Copilot-Userlizenz abgedeckt; ihre Nutzung wird über Tenant-Kapazität oder PAYG gemessen.		 Für Websites, mobile oder eigene Apps sowie andere externe Kanäle. Anonyme oder externe Benutzer werden über Copilot Credits abgerechnet. Teams ist ein Microsoft-365-Kanal und gehört nicht in diese Kategorie.
Eigenes Modell / eigener Orchestrator / eigenes Hosting Custom Engine Agent Azure-Kosten, ggf. Credits
Modell, Orchestrator und Hosting (z.B. Microsoft Foundry, App Service, Bot Service) werden separat abgerechnet. Bei unlizenzierter Nutzung gemeinsamer Tenant-Daten können zusätzlich Copilot Credits anfallen.		 Wenn eigene Modelle, eigene Orchestrierung oder eigenes Hosting nötig sind. Die Gesamtkosten hängen von Hosting, Modell-Tokens, verwendeten Microsoft-365-Diensten und der End-User-Lizenz ab.
