Welches Werkzeug fuer welchen Use Case — verifizierte Grenzen & Hinweise

Use Case Empfohlenes Werkzeug Verifizierte Grenze / Hinweis
Nur Web-Recherche, Drafting – keine Work-/Tenant-Daten M365 Copilot Chat (ohne Lizenz) Geeignet fuer oeffentliche Web-Recherche und Drafting. Fuer Work-/Graph-Grounding oder Zugriff auf Tenant-Daten braucht es eine M365-Copilot-Lizenz oder eine PAYG-/Agent-Konfiguration.
Tiefe Web-/Work-Recherche mit Microsoft Researcher Researcher-Agent / M365 Copilot Researcher ist eine Capability innerhalb von Microsoft 365 Copilot. Sichtbarkeit haengt von Lizenzierung, Rollout und Admin-Konfiguration ab.
Copilot direkt auf SharePoint-Site / Dokumenten-Q&A SharePoint Agent (ready-made) Jede SharePoint-Site hat einen ready-made Agent auf Basis der Site-Inhalte. Nutzung erfordert M365 Copilot Lizenz oder aktivierten Pay-as-you-go-Service.
SharePoint-Agent mit eigener Quellen-Auswahl SharePoint Agent (custom) Bis zu 20 Source Items (Sites, Dokumentbibliotheken, Ordner oder Dateien). Bei mehr als 20 Quellen hoeheres Nesting nutzen – ganze Site/Library statt einzelner Dateien.
Mehrere Sites / groessere Wissensbasis / viele Knowledge Sources Copilot Studio Bis zu 500 Knowledge Sources pro Agent ueber alle Typen hinweg.
Eine einzelne SharePoint-/Microsoft-Liste als Knowledge Source Copilot Studio nicht SharePoint Agent Korrigiert: SharePoint Agents nutzen gemaess offizieller Microsoft-FAQ aktuell keine Daten aus Listen. Auch fuer eine einzelne Liste muss Copilot Studio verwendet werden.
Mehrere SharePoint-Listen Copilot Studio Bis zu 15 Listen pro Agent (empfohlen max. 10). Max. 35'000 Rows pro Liste und insgesamt max. 120'000 Rows; groessere Listen beeintraechtigen Qualitaet und Latenz.
Analytische Fragen ueber strukturierte Listendaten Copilot Studio + SharePoint Lists Knowledge Geeignet fuer Durchschnittswerte, Aggregationen, Counts oder strukturierte Abfragen. Hinweis List-Abfragen beruecksichtigen nur die ersten 2'048 Zeilen und Agenten koennen keinen Code ausfuehren – bei grossen Listen koennen Antworten ungenau sein.
Filtern, OData-Abfragen, gezielte Queries, Schreiben in Listen/Systeme Copilot Studio + Actions / Agent Flow / Power Automate / Connector Fuer gezielte Abfragen, Business-Logik, Writebacks oder CRUD-Szenarien Actions, Agent Flows, Power Automate oder Connectoren nutzen. Knowledge Sources allein sind dafuer nicht der richtige Mechanismus.
Persoenliche Files/Folders OneDrive Agent Fuer persoenliche Dateien und Ordner in OneDrive. Der Agent wird als .agent gespeichert und nutzt die ausgewaehlten Files/Folders als Wissensbasis.
Teams / Outlook / Web / externe Daten als Knowledge M365 Copilot Agent Builder / Declarative Agent Fuer einfache, deklarative Agenten innerhalb von M365 Copilot – Knowledge aus SharePoint, Teams, Outlook, oeffentlichen Websites oder externen Daten via M365 Copilot Connectors.
Cross-System, Dataverse, Power Automate, APIs, komplexe Workflows Copilot Studio (full) Richtige Wahl, wenn der Agent ueber mehrere Systeme arbeitet, Actions ausfuehrt, Dataverse/Power Automate/API-Integrationen benoetigt oder produktiv fuer groessere Zielgruppen bereitgestellt wird.
Autonome oder event-getriggerte Agents Copilot Studio / Custom Engine Agent Fuer automatische Aktionen, Trigger, Agent Flows und komplexe Automatisierungen. Billing und Credits separat gemaess aktueller Microsoft-Dokumentation pruefen.
Externe Website- / App-Kanaele Copilot Studio Wenn der Agent ausserhalb von Microsoft 365 bereitgestellt wird (Websites, Apps, Teams, andere Kanaele). Lizenzierung und Kanalverfuegbarkeit separat pruefen.
Eigenes Modell / eigener Orchestrator / eigenes Hosting Custom Engine Agent Wenn eigene Modelle, eigene Orchestrierung oder eigenes Hosting noetig sind. Diese Agents laufen ausserhalb der Standard-M365-Agent-Runtime.
