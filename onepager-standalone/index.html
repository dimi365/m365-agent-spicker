Welches Werkzeug für welchen Use Case — verifizierte Grenzen & Hinweise

Use Case Empfohlenes Werkzeug Verifizierte Grenze / Hinweis
Nur Web-Recherche, Drafting – keine Work-/Tenant-Daten M365 Copilot Chat (ohne Lizenz) Geeignet für öffentliche Web-Recherche und Drafting. Für Work-/Graph-Grounding oder Zugriff auf Tenant-Daten braucht es eine M365-Copilot-Lizenz oder eine PAYG-/Agent-Konfiguration.
Tiefe Web-/Work-Recherche mit Microsoft Researcher Researcher-Agent / M365 Copilot Researcher ist eine Capability innerhalb von Microsoft 365 Copilot. Sichtbarkeit hängt von Lizenzierung, Rollout und Admin-Konfiguration ab.
Copilot direkt auf SharePoint-Site / Dokumenten-Q&A SharePoint Agent (ready-made) Jede SharePoint-Site hat einen ready-made Agent auf Basis der Site-Inhalte. Nutzung erfordert M365 Copilot Lizenz oder aktivierten Pay-as-you-go-Service.
SharePoint-Agent mit eigener Quellen-Auswahl SharePoint Agent (custom) Bis zu 20 Source Items (Sites, Dokumentbibliotheken, Ordner oder Dateien). Bei mehr als 20 Quellen höheres Nesting nutzen – ganze Site/Library statt einzelner Dateien.
Mehrere Sites / grössere Wissensbasis / viele Knowledge Sources Copilot Studio Bis zu 500 Knowledge Sources pro Agent über alle Typen hinweg.
Eine einzelne SharePoint-/Microsoft-Liste als Knowledge Source Copilot Studio nicht SharePoint Agent Korrigiert: SharePoint Agents nutzen gemäss offizieller Microsoft-FAQ aktuell keine Daten aus Listen. Auch für eine einzelne Liste muss Copilot Studio verwendet werden.
Mehrere SharePoint-Listen Copilot Studio Bis zu 15 Listen pro Agent (empfohlen max. 10). Max. 35’000 Rows pro Liste und insgesamt max. 120’000 Rows; grössere Listen beeinträchtigen Qualität und Latenz.
Analytische Fragen über strukturierte Listendaten Copilot Studio + SharePoint Lists Knowledge Geeignet für Durchschnittswerte, Aggregationen, Counts oder strukturierte Abfragen. Hinweis List-Abfragen berücksichtigen nur die ersten 2’048 Zeilen und Agenten können keinen Code ausführen – bei grossen Listen können Antworten ungenau sein.
Filtern, OData-Abfragen, gezielte Queries, Schreiben in Listen/Systeme Copilot Studio + Actions / Agent Flow / Power Automate / Connector Für gezielte Abfragen, Business-Logik, Writebacks oder CRUD-Szenarien Actions, Agent Flows, Power Automate oder Connectoren nutzen. Knowledge Sources allein sind dafür nicht der richtige Mechanismus.
Persönliche Files/Folders OneDrive Agent Für persönliche Dateien und Ordner in OneDrive. Der Agent wird als .agent gespeichert und nutzt die ausgewählten Files/Folders als Wissensbasis.
Teams / Outlook / Web / externe Daten als Knowledge M365 Copilot Agent Builder / Declarative Agent Für einfache, deklarative Agenten innerhalb von M365 Copilot – Knowledge aus SharePoint, Teams, Outlook, öffentlichen Websites oder externen Daten via M365 Copilot Connectors.
Cross-System, Dataverse, Power Automate, APIs, komplexe Workflows Copilot Studio (full) Richtige Wahl, wenn der Agent über mehrere Systeme arbeitet, Actions ausführt, Dataverse/Power Automate/API-Integrationen benötigt oder produktiv für grössere Zielgruppen bereitgestellt wird.
Autonome oder event-getriggerte Agents Copilot Studio / Custom Engine Agent Für automatische Aktionen, Trigger, Agent Flows und komplexe Automatisierungen. Billing und Credits separat gemäss aktueller Microsoft-Dokumentation prüfen.
Externe Website- / App-Kanäle Copilot Studio Wenn der Agent ausserhalb von Microsoft 365 bereitgestellt wird (Websites, Apps, Teams, andere Kanäle). Lizenzierung und Kanalverfügbarkeit separat prüfen.
Eigenes Modell / eigener Orchestrator / eigenes Hosting Custom Engine Agent Wenn eigene Modelle, eigene Orchestrierung oder eigenes Hosting nötig sind. Diese Agents laufen ausserhalb der Standard-M365-Agent-Runtime.
